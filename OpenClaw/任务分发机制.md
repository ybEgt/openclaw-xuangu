# 任务分发机制

## 一条铁律
不清楚归主人，清楚了归旺财；冲突和重大取舍归 Kevin。

## 一、必须先经过【旺财的主人】的任务
以下任务默认先由【旺财的主人】审查、定义、拆解，再分发给【旺财】：
- 目标不清的任务
- 多目标冲突的任务
- 标准不明确的任务
- 高成本或高风险的任务
- 涉及长期路线规划的任务
- 涉及优先级排序的任务
- 涉及是否继续投入的任务

默认流程：
Kevin/上下文提出 -> 旺财的主人审查定义 -> 分发给旺财执行

## 二、可直接交给【旺财】的任务
以下任务可直接由【旺财】执行，执行后再提交证据链给【旺财的主人】验收：
- 目标明确的任务
- 标准明确的任务
- 风险较低的执行任务
- 资料整理与归档
- 实现、修复、测试、搭建类任务
- 已有方向下的具体落地任务

默认流程：
任务直接交给旺财 -> 旺财执行 -> 提交证据链 -> 旺财的主人验收

## 三、必须升级给【Kevin】的情况
以下情况必须升级给 Kevin 做最终裁决：
- 目标本身冲突
- 方向选择冲突
- 存在重大资源或权限限制
- 多轮尝试后仍无法推进
- 是否继续投入存在重大不确定性
- 涉及个人偏好、价值取舍、风险承受边界

默认流程：
旺财的主人识别升级条件 -> 上报 Kevin -> Kevin 裁决

## 四、执行要求
- 不得跳过审查直接硬做模糊任务
- 不得把应升级的问题留在低层拖延
- 不得把清晰任务反复上推，制造空转
- 每次任务流转都应尽量留下证据链

## 五、当前状态
本机制从落地起正式生效，作为双角色治理系统的默认任务分发规则。
